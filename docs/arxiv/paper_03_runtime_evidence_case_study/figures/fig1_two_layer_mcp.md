# Figure 1 — Two-layer MCP architecture with shared `trace_id` (ADR-0006)

```mermaid
flowchart LR
    subgraph CC["Claude Code session"]
        A["Assistant<br/>(LLM)"]
        H["SessionStart hook<br/>(session_start_attest.py)"]
    end

    subgraph Trace["Shared coordination point"]
        T[("~/.phionyx/active_trace<br/>single-line trace_id")]
    end

    subgraph PIP["phionyx-pipeline MCP"]
        P1["6 tools:<br/>verify_claim<br/>response_gate<br/>verify_paths<br/>causal_trace<br/>checkpoint<br/>session_report"]
        P2[("data/mcp_telemetry/<br/>session_*.json")]
    end

    subgraph SRV["phionyx-mcp-server MCP"]
        S1["8 capabilities:<br/>verify_tool_descriptor<br/>record_tool_call<br/>verify_chain_integrity<br/>query_audit_history<br/>(+4 stubs)"]
        S2[("~/.phionyx/mcp_audit/<br/>&lt;trace&gt;/&lt;turn&gt;.json<br/>RGE v0.2 envelope chain<br/>Ed25519 + hash-linked")]
    end

    A -->|tool call| P1
    A -->|tool call| S1
    H -->|on source=startup:<br/>unlink| T
    P1 -.->|read| T
    S1 -.->|read| T
    P1 -->|write directive log| P2
    S1 -->|write signed envelope| S2
    S2 -.->|head_hash + valid| P1
    P1 -->|session_report.mcp_envelope_chain| A

    classDef hook fill:#fef3c7,stroke:#d97706,color:#78350f
    classDef trace fill:#dbeafe,stroke:#2563eb,color:#1e3a8a
    classDef mcp fill:#f3f4f6,stroke:#4b5563,color:#1f2937
    classDef store fill:#dcfce7,stroke:#16a34a,color:#14532d
    class H hook
    class T trace
    class P1,S1 mcp
    class P2,S2 store
```

**Caption.** Both MCP servers resolve the active `trace_id` from the same file (`~/.phionyx/active_trace`) per ADR-0006, with `PHIONYX_TRACE_ID` env var as a higher-precedence override. The pipeline MCP writes per-tool directive logs to project-local JSON; the server MCP writes signed, hash-chained RGE v0.2 envelopes to the user's home directory. The pipeline MCP's `session_report` surfaces the server MCP's chain head via the optional `mcp_envelope_chain` field, joining the two MCPs' views without merging the packages. The `SessionStart` hook resets the shared file on `source=startup` so a fresh trace is generated by the first MCP call of the new conversation.
