# Phionyx v0.6.0 — Multi-Agent Evidence

**Release date:** 2026-05-25 (target was 2026-09-29 → shipped ~4 months early).
**Theme:** Parent / child trace + subagent handoff + replay across multi-agent setups; cross-runtime evidence portability.

This release answers the next question after *a model saying fixed is not evidence*: when multiple agents collaborate, what shape lets a reviewer reconstruct the flow from the signed evidence alone, with no insider access and no trust in any single agent's narration?

Four features ship: the multi-agent envelope chain (F5), an LLM-as-judge eval primitive (F9), cross-runtime evidence portability for Langfuse / LangSmith (F13), and the F12-plus extension to the Claude Code plugin from v0.5.1. Plus the v0.6 schema portfolio is published as a stabilisation candidate.

## F5 — Multi-agent / subagent audit chain

The `subagent_chain` block reserved in RGE v0.2 §2.2.3 as `reserved-for-v0.6.0-f5` is now an active specification.

**Surface:**
- `SubagentChainV0` Pydantic v2 model in `phionyx_core.contracts.envelopes` — `agent_id`, `role` ∈ {root, child, leaf}, `chain_depth`, `protocol` ∈ {a2a, agntcy, phionyx_native, langgraph_subgraph, crewai, autogen}, `parent_envelope_hash`, `parent_agent_id`, `child_agent_ids`, `handoff_signature`, `handoff_timestamp_utc`, `protocol_data`.
- `compute_handoff_signing_body()` — canonical-JSON encoding (sort_keys, no whitespace, ASCII-safe) of the 5-field handoff signing body per RFC §2.4.
- JSON Schema (Draft 2020-12) at `examples/envelopes/subagent_chain_v0_1/subagent_chain_v0_1.schema.json` — 8 required + 3 optional fields; if/then invariants enforce root depth=0 + null parent fields, child/leaf depth≥1 + non-null parent fields, leaf has zero child_agent_ids.
- LangGraph supervisor adapter in `phionyx-langchain-langgraph` populates the block on every `register()` and `handoff()` emit. Compatible with v0.5.0 envelope shape: the block is optional and additive.

**Worked examples:**
- LangGraph 3-agent researcher → writer → editor chain — `subagent_chain_v0_1_minimal_envelope.json`.
- A2A protocol-mapped 3-agent chain (research-agent → drafting-agent → review-agent) — `subagent_chain_v0_1_a2a_envelope.json`.

**Tests:** 49 new (12 contract `tests/contract/test_subagent_chain_schema.py` + 30 unit `tests/unit/core/test_subagent_chain.py` + 7 integration `tools/phionyx_langchain_langgraph/tests/test_multi_agent_chain.py`). Zero regression to existing test suites (125/125 contract, 28/28 envelope unit, 52/52 langgraph preserved).

**Decision record:** [ADR-0007](../adr/0007-multi-agent-envelope-chain.md) — Multi-Agent Envelope Chain.

## F9 — LLM-as-judge eval module

New companion package `phionyx-eval` at `tools/phionyx_eval/`. Public repo (`halvrenofviryel/phionyx-eval`) + PyPI publication via trusted-publisher pattern follows on a separate ship.

**Surface:**
- `LLMClient` Protocol — caller supplies any client with `.complete(prompt: str) -> str`. No provider SDK pin.
- `Rubric`, `Judgment`, `JudgmentScore` Pydantic v2 models with frozen rubrics and validated scoring scales.
- `LLMAsJudge` — builds structured JSON-only prompt, parses response with markdown-fence tolerance + 9 distinct malformed-response rejection paths.
- 4 standard rubrics (`EVIDENCE_COVERAGE`, `CORRECTNESS`, `COMPLETENESS`, `INDEPENDENT_VERIFIABILITY`) on a 0–5 integer scale.
- Deterministic verdict derivation: aggregate score normalised to [0, 1]; `pass` if ≥ threshold, `uncertain` if within 0.05 below threshold, `fail` otherwise. The runtime votes on verdicts; the model only votes on scores.
- `build_judgment_envelope` mirrors the audit-chain pattern from `phionyx-langchain-langgraph` — schema `phionyx.judgment_envelope.v1`, hash-chained, HMAC-signed by default (Ed25519 swap-in).

**Tests:** 39 (12 rubric + 17 judge + 10 envelope chain). End-to-end quickstart runnable without an API key (canned FakeClient).

## F13 — Cross-runtime evidence portability

Import-only: read Langfuse traces and LangSmith run trees, emit Phionyx-shaped signed envelope chains. Round-trip lossless for mappable fields; non-mappable foreign fields preserved verbatim under `subject.metadata.imported_extras`.

**Surface:**
- `import_langfuse_trace(trace, *, signer=None)` — schema `phionyx.imported_langfuse_envelope.v1`. Maps 13 trace-level + 12 observation-level fields. Schema-marked provenance lets consumers dispatch on source without re-parsing.
- `import_langsmith_run(run, *, descendants=None, signer=None)` — schema `phionyx.imported_langsmith_envelope.v1`. Depth-first pre-order traversal of the run tree; missing descendants skipped silently to tolerate truncated exports.
- `ImportResult` + `MappingReport` dataclasses surface what mapped, what was preserved as `imported_extras`, what was dropped.

**Tests:** 42 (21 Langfuse + 21 LangSmith). Two fixtures (`tests/fixtures/langfuse_sample.json` + `langsmith_sample.json`) carry vendor-specific extras to exercise the preservation path.

## F12-plus — Claude Code plugin extension

New command `/phionyx:replay-trace` extends the v0.5.1 Phionyx Claude Code plugin to walk multi-agent envelope chains timestep-by-timestep. Two worked walkthroughs ship: LangGraph supervisor (researcher → writer → editor) and OpenAI Agents SDK (multi-span tree with siblings).

**Wire path:** Plugin reads envelope chains via existing `phionyx-pipeline-mcp` + `phionyx-mcp-server` surfaces (no new MCP tools). The command interprets and presents persisted envelopes; it does not re-run agents and does not modify state.

**Tests:** 7 plugin manifest + smoke tests (existing v0.5.1 suite, extended with required-file assertions for the new replay-trace command + walkthroughs).

## What is NOT in v0.6.0

- **Cross-runtime export.** v0.6.0 W3 ships import-only. Phionyx → Langfuse / LangSmith export is candidate work for v0.7.0.
- **OpenAI Agents `subagent_chain` integration.** The OpenAI Agents bridge (`phionyx-openai-agents`) does not yet populate the block. Demand-driven add in v0.7.0 or v0.8.0.
- **AGNTCY/ACP worked example.** The v0.1 RFC names the protocol mapping but ships no worked example. Deferred to v0.6.x patch if adoption signals justify.
- **Full A2A protocol adapter.** The A2A worked example demonstrates the wire shape; the full A2A protocol-level signature integration is v1.1 scope per the public roadmap.
- **`phionyx-eval` PyPI publication.** Ships in this release as `tools/phionyx_eval/` in the monorepo. Public-repo promotion (`halvrenofviryel/phionyx-eval`) + PyPI publication follows on a separate ship via the same trusted-publisher pattern used for F3/F11/F10.

## Migration

v0.6.0 is additive. Existing v0.5.x producers continue to work unchanged. Adopting new v0.6.0 surfaces is opt-in:

- Adopt `subagent_chain` v0.1 by following [`examples/envelopes/subagent_chain_v0_1/migration_rge_v0_2_to_subagent_v0_1.md`](../../examples/envelopes/subagent_chain_v0_1/migration_rge_v0_2_to_subagent_v0_1.md).
- Adopt `phionyx-eval` by `pip install phionyx-eval` once the package publishes (use monorepo path `tools/phionyx_eval/` until then).
- The Claude Code plugin's `/phionyx:replay-trace` is available wherever the v0.5.1 plugin is installed; no upgrade action needed beyond pulling latest.
